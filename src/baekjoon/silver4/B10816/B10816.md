# BOJ 10816 - 숫자 카드 2

## 문제 설명
N개의 숫자 카드가 주어지고, M개의 숫자에 대해 각각 해당 숫자가 몇 개 존재하는지 출력하는 문제이다.

---

## 문제 풀이 과정

처음에는 배열과 반복문을 이용하여 문제를 해결하려고 했다. 각 숫자마다 전체 배열을 순회하면서 개수를 세는 방식으로 구현했는데, 이 경우 반복문이 중첩되면서 시간복잡도가 커졌고 실제로 제출했을 때 시간 초과가 발생하였다.

이후 ArrayList를 사용하여 다시 구현해보았다. 리스트에 값을 저장한 뒤, 각 숫자마다 순차적으로 비교하며 개수를 세는 방식으로 작성했지만, 결국 모든 값을 하나씩 확인해야 한다는 점에서 근본적인 시간복잡도 문제는 해결되지 않았다. 이 과정을 통해 단순히 자료구조만 바꾸는 것으로는 성능이 개선되지 않는다는 것을 느꼈다.

그래서 탐색을 더 빠르게 할 수 있는 방법을 고민하게 되었고, 해시 기반 자료구조를 사용해보기로 했다. HashMap을 이용하여 각 숫자를 key로, 해당 숫자의 개수를 value로 저장하도록 구현하였다. 이후에는 원하는 숫자를 바로 조회하여 결과를 출력할 수 있었고, 이 방식으로 시간 초과 문제를 해결할 수 있었다.

이 문제를 통해 단순 구현보다 입력 크기와 시간복잡도를 먼저 고려하는 것이 중요하다는 것을 느꼈고, 상황에 맞는 자료구조를 선택하면 코드의 효율성과 가독성을 모두 개선할 수 있다는 점을 체감할 수 있었다.
