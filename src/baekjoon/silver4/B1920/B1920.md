# BOJ 1920 - 수 찾기

## 문제 설명
N개의 정수가 주어지고, M개의 정수가 각각 N에 포함되어 있는지를 판단하여 존재하면 1, 아니면 0을 출력하는 문제이다.

## 문제 풀이 과정

처음에는 배열을 이용하여 문제를 해결하였다.  
N개의 수를 배열에 저장한 뒤, M개의 수 각각에 대해 배열을 처음부터 끝까지 순차적으로 탐색하면서 값이 존재하는지를 확인하는 방식으로 구현하였다. 이 방식은 직관적이지만 모든 값을 하나씩 비교해야 하기 때문에 반복문이 중첩되는 구조가 되었고, 실제로 구현하는 데에도 생각보다 시간이 걸렸다.

이 상태로 제출했을 때 시간 초과가 발생하였고, 단순 반복으로는 해결하기 어렵다고 판단하였다. 그래서 탐색 시간을 줄일 수 있는 방법을 고민하였고, `HashSet`을 사용하여 다시 구현하였다.

N개의 수를 `HashSet`에 저장한 뒤, M개의 수에 대해 `contains()`로 존재 여부를 확인하는 방식으로 바꾸었더니 훨씬 간단하게 구현할 수 있었고, 시간 초과 문제도 해결할 수 있었다.

이번 문제를 통해 단순히 구현하는 것보다 입력 크기를 고려한 접근이 중요하다는 것을 느꼈고, 상황에 맞는 자료구조를 선택하면 코드의 복잡도와 실행 시간 모두를 줄일 수 있다는 것을 직접 체감할 수 있었다.
